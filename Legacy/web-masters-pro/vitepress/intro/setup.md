# Установка

Для начала работы с проектом необходимо выполнить несколько шагов по установке и настройке необходимых компонентов.

## Требования к системе

Перед началом убедитесь, что у вас установлены следующие программы:

- Docker: Версия 27.x или выше
- Docker Compose: Версия 2.31.x или выше
- Node.js: Версия 22.x или выше
- Composer: Версия 2.8.x или выше

## Установка зависимостей

### Установка Docker

Если у вас еще не установлен Docker, следуйте инструкциям на официальном сайте Docker для вашей операционной системы:

- Для Windows: https://docs.docker.com/docker-for-windows/install/
- Для macOS: https://docs.docker.com/docker-for-mac/install/
- Для Linux: https://docs.docker.com/engine/install/

После установки проверьте корректность работы командой:

````shell
docker --version
````

Для запуска контейнеров выполните команду:

````shell
docker-compose up
````

::: info 
Установка пакетов локально (описанная ниже) является необязательной процедурой. Все необходимые для работы пакеты автоматически устанавливаются непосредственно в контейнеры во время их сборки.
:::

### Локальная установка пакетов Composer для Laravel

Перейдите в папку `laravel` и выполните команду: 

````shell
composer install
````

### Локальная установка пакетов npm для Nuxt

Перейдите в папку `nuxt` и выполните команду: 

````shell
npm install
````

### Локальная установка пакетов npm для VitePress

Перейдите в папку `vitepress` и выполните команду: 

````shell
npm install
````

## Настройка окружения

### Создание .env файла

Скопируйте файл `.env.example` в корневой директории проекта и переименуйте его в `.env`. Откройте этот файл и внесите изменения согласно вашим требованиям.

### Настройка базы данных

Убедитесь, что в файле `.env` указаны правильные параметры для подключения к базе данных MySQL:

- `MYSQL_ROOT_PASSWORD`: Пароль для root-пользователя MySQL
- `MYSQL_DATABASE`: Имя базы данных
- `MYSQL_USER`: Пользователь базы данных
- `MYSQL_PASSWORD`: Пароль пользователя базы данных

### Указание домена проекта

Задайте доменное имя для вашего проекта в переменной `APP_DOMAIN` в файле `.env`.

Для локального тестирования добавьте запись в ваш файл `hosts` (или аналогичный файл на вашей ОС):

````shell
127.0.0.1 docker.local
````

Теперь ваш проект будет доступен по адресу `http://docker.local`.
