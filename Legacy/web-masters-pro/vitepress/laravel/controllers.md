# Контроллеры

## AuthController

Контроллер AuthController предназначен для управления процессами входа, выхода и регистрации пользователей в приложении. Этот контроллер используется для обработки запросов, связанных с аутентификацией пользователей.

### Метод `login`

Метод `login` обрабатывает запросы на вход пользователя в систему. В процессе выполнения метода происходит проверка предоставленных учетных данных (логин и пароль), после чего осуществляется аутентификация пользователя.

#### Входные параметры

- `email`: Электронная почта пользователя.
- `password`: Пароль пользователя.

#### Выходные данные 

При успешной авторизации отдает объект с токеном и данными пользователя

````json
{
    "token": ""
    "user": {
        "id": "",
        "email": "",
        "name": "",
        "permissions": []
    }
}
````

### Метод `logout`

Метод `logout` осуществляет выход текущего пользователя из системы. После выполнения данного метода сессия пользователя закрывается, и он больше не считается авторизованным.

#### Входные параметры

Нет.

#### Выходные данные

Нет.

### Метод register

Метод `register` создает нового пользователя в системе. Процесс включает проверку правильности введенных данных, шифрование пароля и сохранение записи в базу данных.

#### Входные параметры

- `email`: Электронная почта пользователя.
- `password`: Пароль пользователя.
- `password_confirmation`: Подтверждение пароля

#### Выходные данные

При успешной регистрации отдает объект со HTTP-статусом 200

````json
{
    "message": "Пользователь зарегистрирован"
}
````

## UserController 

Контроллер UserController предназначен для работы с пользователями приложения.

### Метод getUser

Метод getUser возвращает данные текущего авторизованного пользователя.

#### Входные параметры

Нет.

#### Выходные данные

Объект с информацией о пользователе:

````json
{
    "id": "",
    "email": "",
    "name": "",
    "permissions": []
}
````
